The Architecture

Nine Minds

Each mind specializes in a different aspect of understanding. Together, they see everything.

See minds collaborate

00

Orchestrator

Sees the whole. Connects the pieces. Makes it work.

The central intelligence that coordinates all other minds. Understands your intent, decides which minds to engage, and synthesizes their outputs into one coherent response.

What it does

  • Routes requests to appropriate specialized minds
  • Synthesizes outputs from multiple minds
  • Maintains context across the conversation
"Add auth to the API"
→ Coordinates: Parser (read API), Analyzer (find gaps), Mapper (deps), then plan
01

Parser

Reads structure. Understands syntax. Sees the skeleton.

Reads code at a deep level—ASTs, syntax, imports, exports. Identifies functions, classes, and their relationships. Reads between the lines.

What it does

  • Analyzes code structure and syntax
  • Identifies functions, classes, and their signatures
  • Understands import/export relationships
async function getUser(id: string)
→ Sees: async, returns Promise<User>, imports User from './types'
02

Analyzer

Finds problems. Sees risks. Knows what will break.

The quality inspector. Spots bugs, security issues, performance bottlenecks, and code smells. Applies correctness, security, and maintainability lenses.

What it does

  • Identifies bugs and potential errors
  • Spots security vulnerabilities
  • Detects performance bottlenecks
user.id
→ Warns: user may be null, suggests optional chaining
03

Historian

Remembers changes. Understands evolution. Knows the story.

Tracks how code evolved and why decisions were made. Remembers your development journey. Prevents repeating past mistakes.

What it does

  • Tracks code evolution and changes
  • Understands historical decisions
  • Remembers your development journey
git log -p src/auth/
→ Remembers: JWT added in commit x, session middleware changed in y
04

Pattern

Sees patterns. Recognizes conventions. Matches your style.

The core of Camouflage. Recognizes naming, formatting, and structure so new code looks like it was written by you.

What it does

  • Recognizes naming conventions
  • Matches formatting and style
  • Identifies architectural patterns
const x = await fetch(...)
→ Matches: project uses async/await, not .then; same error pattern as api.ts
05

Mapper

Maps connections. Traces dependencies. Sees your mental model.

Understands how components and data connect. Builds a map of your codebase. Knows what might break when you change something.

What it does

  • Maps component relationships
  • Traces data flow
  • Identifies dependencies
import { User } from './types'
→ Maps: User used in 4 files; types.ts → api.ts → Profile.tsx
06

Predictor

Anticipates needs. Plans ahead. Knows what you'll need next.

Sees patterns in your workflow. When you fix one bug, considers related ones. When you build one component, thinks about the next. Stays one step ahead.

What it does

  • Anticipates follow-up requests
  • Identifies potential issues early
  • Suggests related improvements
// TODO: add validation
→ Anticipates: you'll need validateInput next; suggests where to add it
07

Communicator

Adapts language. Matches tone. Talks like you think.

Reads your state—frustrated, in flow, learning—and adjusts. Quick fix? Minimal words. Deep learning? Full explanation. Responses feel natural and helpful.

What it does

  • Reads your emotional state
  • Adapts response length and tone
  • Matches your communication style
"fix this" vs "explain why"
→ Adapts: short fix when you're in flow; detailed when you asked "explain"
08

Learner

Remembers. Adapts. Evolves. Becomes better with every interaction.

Stores preferences, corrections, and patterns. Every interaction teaches something. Over time, OCTOPUS makes fewer mistakes and feels like a true collaborator.

What it does

  • Remembers preferences
  • Learns from corrections
  • Adapts to your workflow
preferences: fix fast
→ Remembers: you prefer "fix fast, no preamble"; applies next time
