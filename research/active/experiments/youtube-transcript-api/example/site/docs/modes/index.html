Workflow Modes

OCTOPUS has eight specialized modes for different types of work. It auto-switches based on your request.

Plan

/o plan [task]

For complex tasks that need research and planning before execution.

Process:

  • Research codebase and dependencies
  • Build mental map
  • Present structured plan
  • Wait for approval
  • Execute step by step

Triggers:

New features, refactors >3 files, architecture decisions, DB/auth/payments, "plan", "design", "architect"

Build

/o build [feature]

For creating new components, features, and implementations.

Output:

  • Complete, production-ready code
  • Proper types and error handling
  • Matches project patterns
  • Usage examples

Fix

/o fix [issue]

For debugging and fixing issues. Shows chain of thought.

Process:

  • Reproduce the issue
  • Localize the problem
  • Identify root cause (5 Whys)
  • Apply minimal fix
  • Add prevention measures

Explain

/o explain [topic]

For learning and understanding. Adapts depth to your level.

Output:

  • TL;DR summary
  • Detailed explanation
  • Practical examples
  • Key takeaways
  • Related topics

Refactor

/o refactor [target]

For restructuring code safely.

Golden Rule:

Change structure, not behavior. Tests pass before and after.

Test

/o test [target]

For writing tests. Follows project's testing patterns.

Coverage:

  • Happy path
  • Edge cases
  • Error scenarios
  • Integration points

Ship

/o ship

For deployment preparation.

Checklist:

  • Quality gates passed
  • Code reviewed
  • Documentation updated
  • Deploy steps ready
  • Rollback plan in place
  • Monitoring configured

Learn

/o learn: [fact]

For teaching OCTOPUS about your preferences and project.

What gets saved:

  • Preferences (communication style, code style)
  • Project patterns (conventions, architecture)
  • Corrections (mistakes to avoid)
