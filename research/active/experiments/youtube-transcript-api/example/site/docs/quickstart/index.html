Quick Start

Get OCTOPUS running in your Cursor workspace in under 5 minutes.

The octopus has no shell… so it developed intelligence.

Prerequisites

  • Cursor IDE installed
  • A project folder to work with
  • Basic familiarity with command-line interfaces

Installation

Step 1: Create .cursorrules

In your project root, create a file named .cursorrules:

.cursorrules 
# OCTOPUS - Nine Minds. One Purpose.
# This file tells Cursor to load OCTOPUS protocols.

alwaysApply: true

Step 2: Add the .cursor folder

Create a .cursor folder with the required structure:

terminal 
mkdir -p .cursor/rules .cursor/commands .cursor/learnings

Step 3: Copy the octopus-cursor package

Copy the octopus-cursor package into your project root: get it from the repo (clone or download ZIP), then copy octopus-cursor/.cursorrules and octopus-cursor/.cursor/ into your project root. See the full installation guide for details.

Step 4: Verify installation

Open Cursor chat and type:

command 
/o?

You should see OCTOPUS initialize with "All 9 Arms Active".

Tip: If you don't see the expected response, make sure Cursor has reloaded your project. Try closing and reopening the workspace.

Your First Commands

Now that OCTOPUS is installed, try these commands:

Plan a feature

/o plan user authentication with OAuth

OCTOPUS will research your codebase and present a structured plan.

Fix a bug

/o fix the TypeError in UserService

OCTOPUS will diagnose, explain, and fix the issue.

Build a component

/o build a Modal component

OCTOPUS will create production-ready code matching your project's style.
