/d

Design Intelligence

Extract design DNA, create pages and components, check consistency, and adapt styles.

When to use this command

✓ Use /d when:

  • Scanning or extracting design (colors, typography, spacing)
  • Creating a page, component, or feature from the Design Genome
  • Checking design consistency
  • Adapting style (e.g. darker, more rounded)

× Don't use /d when:

  • General coding, building logic, or fixing bugs (use /o)
  • Full codebase review (use /r)

Subcommands

/d scan

Extract the Design Genome from your codebase. Analyzes colors, typography, spacing, motion, and patterns.

Output: Complete design token documentation

/d page [type] [name]

Create a new page matching your design system.

Types: landing, dashboard, form, list, detail, settings

Example: /d page dashboard UserDashboard

/d component [name]

Create a component following your patterns. Includes proper types, props, styling.

Example: /d component Button

/d feature [name]

Create a complete feature: types, service, hook, components, route, tests.

Example: /d feature UserProfile

/d check

Validate current code against the Design Genome. Reports inconsistencies.

/d adapt [style]

Modify the design feel while maintaining consistency.

Examples: /d adapt darker, /d adapt more rounded

Design Genome

The Design Genome is OCTOPUS's understanding of your visual system:

  • Colors: Primary, secondary, backgrounds, text, accents
  • Typography: Fonts, weights, sizes, line heights
  • Spacing: Scale, rhythm, padding, margins
  • Borders: Radii, widths, colors
  • Shadows: Elevation system
  • Motion: Durations, easings, transitions
  • Components: Patterns, variants, states

Examples

/d scan
/d page landing Homepage
/d page dashboard AdminDashboard
/d component Modal
/d component Card with hover effects
/d feature Notifications
/d check
/d adapt with more contrast
