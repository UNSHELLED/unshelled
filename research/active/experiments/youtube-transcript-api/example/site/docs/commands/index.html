Command Reference

OCTOPUS uses three main commands: /o for workflow, /d for design, and /r for review.

/o

Workflow Orchestration

Plan, build, fix, explain, refactor, test, ship, and learn. The main command for all coding tasks.
/d

Design Intelligence

Scan design DNA, create pages and components, check consistency, adapt styles.
/r

Full Review

8-perspective code review covering correctness, security, performance, and more.

Quick Reference

Command Description
/o Auto-detect mode from context
/o? Deep init: project + learnings, all arms active (Unshelled)
/o plan [task] Plan a complex task
/o build [feature] Build a feature or component
/o fix [issue] Debug and fix an issue
/o explain [topic] Get an explanation
/o refactor [target] Refactor code safely
/o test [target] Write tests
/o ship Deployment checklist
/o learn: [fact] Teach OCTOPUS something
/o! Emergency mode
/d scan Extract design DNA
/d page [type] [name] Create a page
/d component [name] Create a component
/d check Validate design consistency
/r Full 8-perspective review
