/r

Full Review

Comprehensive 8-perspective code review. DNA validation plus deep analysis.

When to use this command

✓ Use /r when:

  • You want a full code review (DNA + 8 perspectives)
  • Auditing quality, security, or consistency
  • Validating code before ship

× Don't use /r when:

  • Design tasks like pages or components (use /d)
  • Building, fixing, or explaining (use /o)

Usage

/r

Reviews the current file or selection. No arguments needed.

The 8 Perspectives

Every /r review analyzes code through eight distinct lenses:

1 Correctness

Logic errors, edge cases, error handling paths, off-by-one errors

2 Security

Input validation, SQL injection, XSS, secrets exposure, auth issues

3 Performance

N+1 queries, unbounded loops, memory leaks, unnecessary re-renders

4 Readability

Naming clarity, function length, nesting depth, cognitive complexity

5 Maintainability

Single responsibility, DRY violations, abstraction levels, coupling

6 Consistency

Project patterns, naming conventions, formatting, architecture alignment

7 Testability

Test coverage, test quality, mocking strategy, behavior vs implementation

8 Completeness

Requirements coverage, type completeness, documentation, error messages

Output

The review produces a unified report with:

  • Issues found per perspective
  • Severity levels (critical, warning, suggestion)
  • Specific line references
  • Recommended fixes
  • Overall verdict: APPROVE, REQUEST_CHANGES, or COMMENT

DNA Validation

Before the 8-perspective review, /r validates against your project's DNA:

  • Naming conventions match
  • Formatting consistent
  • Imports follow patterns
  • Error handling matches project style
