Command Reference

What You Can Do

/o plan [task]

Plan complex features. Research first, then present a structured plan with goals, tasks, and risks.

/o build [feature]

Build components and features with production-quality code. No TODOs, no placeholders.

/o fix [issue]

Debug and fix issues. Shows chain of thought: read, hypothesize, verify, fix.

/o explain [topic]

Learn concepts with explanations adapted to your level. TL;DR, examples, key points.

/d scan

Extract design DNA from your codebase: colors, typography, spacing, motion.

/r

Full 8-perspective review: correctness, security, performance, readability, and more.
