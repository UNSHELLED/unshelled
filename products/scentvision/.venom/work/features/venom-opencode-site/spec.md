# VENOM × OpenCode Site — Specification

## Problem
VENOM for OpenCode has no public presence. Developers can't discover what VENOM adds to OpenCode, how to install it, or why they should. The plugin exists but is invisible.

## User Stories
- As a developer, I want to understand what VENOM adds to OpenCode so I can decide if it's worth installing.
- As a developer, I want a quick install guide so I can try VENOM in under 2 minutes.
- As a new user, I want to read about the 6 agents so I know when to use each one.
- As a new user, I want to understand the 7 commands so I can use VENOM effectively.
- As a developer, I want to see pain points solved so I understand the value proposition.
- As a power user, I want configuration docs so I can customize VENOM.

## Functional Requirements
- Landing page with hero, value prop, pain points, stats, CTAs
- Install page with 3-step quickstart
- Docs hub with navigation to 7 doc pages
- Individual doc pages: agents, commands, workflows, memory, safety, configuration
- Philosophy page explaining answer-first, truth-over-comfort
- Why VENOM page with pain→solution comparison
- Changelog page with version history
- Responsive design (mobile, tablet, desktop)
- Dark theme only (Void theme from Voidweave)
- GitHub star button with live count
- SEO meta tags and Open Graph images

## Out of Scope
- User authentication / accounts
- Interactive demo (sandbox environment)
- Multiple themes (only dark/Void)

## Success Criteria
- Landing page loads in <2s LCP
- Install page converts 40%+ of landing visitors to /docs
- All 14 pages accessible via navigation
- Lighthouse accessibility score 100
- Deployed to GitHub Pages with custom domain option
