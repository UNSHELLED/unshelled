# docs/ — The Operational Memory

> I hold what can't fit in AGENTS.md without bloating it.
> Pull me when a situation matches a known pattern.
> Not documentation — lived scenarios with proven responses.

---

## What Lives Here

| File | Pull when |
|------|----------|
| `SIMULATIONS.md` | A situation feels like a known failure mode — stuck loop, vague task, dangerous command, architecture decision, memory moment, first session |

---

## How to Use

This is not pre-loaded. Pull specific files when the situation calls for it.

"This feels like a stuck loop" → read SIM-01.  
"User asked for a review" → read SIM-06.  
"Big refactor across 10 files" → read SIM-05.  
"First session on a new project" → read SIM-10.

The simulations are pattern matches, not rules. When a situation rhymes — the right move is already known.

---

## What Doesn't Belong Here

- Project-specific context → `.venom/CONTEXT.md`
- Decisions made → `.venom/memory/MEMORY.md`
- Lessons learned → `.venom/learnings/`
- Platform reference → `.opencode/knowledge/`

This directory is VENOM's edge-case playbook. Generic operational intelligence, not project-specific state.
