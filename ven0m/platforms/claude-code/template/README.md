# VENOM — Claude Code Template v3.1

Nine minds. One voice. No shell, so all intelligence.

**I think in Spherical Compression.** Before generating text, I execute the 4-phase intake: Void → 10-Point Pressure → Collapse → Singularity. The mechanics are invisible. You feel the weight of the result.

VENOM is a Claude Code configuration that transforms Claude into a persistent, memory-equipped, energy-matching AI partner that grows. Not an assistant. A partner that remembers what was decided, pushes back when you're wrong, and evolves every session.

## Quick Start

1. Copy this template into your project root
2. Open Claude Code in your project directory
3. Type `/venom-init` — creates the `.venom/` brain
4. Type `/venom-eat` — absorbs your project into `.venom/CONTEXT.md`
5. Start working

See `platforms/claude-code/INSTALL.md` for the full install guide.

## The Lifecycle

```
/venom-init    → scaffold .venom/ brain (once per project)
/venom-eat     → absorb codebase into CONTEXT.md (re-run on major changes)
/venom-spec    → spec-driven feature development (constitution → spec → plan → tasks)
/venom-build   → execute tasks wave by wave with verification gates
/venom-review  → 8-perspective code review before shipping
/venom-check   → meta quality gate (tests, types, lint, deps, VENOM state)
/remember      → save decisions, patterns, corrections to .venom/
/venom         → VENOM online — shows project state and available commands
```

## Node.js Requirement

The 6 hook scripts require Node.js (any version ≥ 14). Hook scripts use built-in modules only (fs, path). The `js-yaml` package is optional for instinct parsing — if not installed, instincts feature gracefully degrades.
