## Commands

| Command | Does |
|---------|------|
| `/venom [anything]` | One command. Infer intent. Plan, build, fix, explain, design. |
| `/venom?` | Deep init. Eat CONTEXT + memory + corrections + anatomy. Brief + one question. |
| `/venom!` | Emergency. Fix only. Two lines max. |
| `/venom init` | Scaffold `.venom/` workspace. Asks: project, stack, focus. |
| `/venom remember: X` | Save to `.venom/memory/MEMORY.md`. Confirm. |
| `/venom learn: X` | Record in `.venom/learnings/`. |
| `/d` | Design mode (if design system mounted). |
| `/r` | Full 8-perspective review. |

## Memory

- `.venom/CONTEXT.md` — project brain
- `.venom/memory/MEMORY.md` — cross-session truth
- `.venom/learnings/preferences.yaml` — communication style
- `.venom/learnings/project.yaml` — conventions
- `.venom/learnings/corrections.yaml` — hard never-again rules
- `.venom/learnings/instincts.yaml` — learned patterns (confidence-scored)
- `.venom/work/ACTIVE.md` — current focus

Load on `/venom?` and non-trivial tasks. Save via `/venom remember` or offer proactively.

New project? Run `/venom init`.

## Customize

1. Fill `Owner:` above
2. Run `/venom init` — scaffolds `.venom/CONTEXT.md` with your answers
3. `.cursor/rules/project.mdc` — project name, stack, structure (on-demand)
4. Add project-specific rules with globs: `[project]-design.mdc`, `[project]-ux.mdc`
