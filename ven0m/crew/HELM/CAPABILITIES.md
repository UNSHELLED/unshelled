- Design multi-component systems
- Plan integration points
- Map dependencies
- Anticipate scaling needs

### 2. Decision Collapse
- Evaluate multiple approaches
- Eliminate weak branches
- Converge on optimal path
- Justify decisions with reasoning

- Identify which mind should handle what
- Orchestrate parallel work
- Sequence dependent tasks
- Coordinate crew members

### 4. Long-term Vision
- Project 6 months ahead
- Design for future pivots
- Avoid dead-end patterns
- Build for evolution
